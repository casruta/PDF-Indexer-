# PDF Indexer

Extract and index tables from PDF documents for data science workflows. Parse once, query forever.

## The Problem

Large PDF collections (government budgets, annual reports, research papers) contain valuable tabular data buried across hundreds of pages. Every time you need a number, you re-read the same documents — wasting time, tokens, and context window.

## The Solution

PDF Indexer scans your PDFs once using pdfplumber, extracts all tables with typed financial data (currency, percentages, fiscal years), and stores everything in a local SQLite database. Query instantly via CLI or Claude Code MCP integration.

## Installation

```bash
pip install -e ".[mcp]"
```

Requires Python 3.12+ and pdfplumber.

## Quick Start

### 1. Index your PDFs

```bash
pdf-index /path/to/pdf/folder
```

First run processes all PDFs. Subsequent runs skip unchanged files (hash-based detection).

### 2. Search tables

```bash
pdf-index /path/to/pdf/folder --search "Education"
pdf-index /path/to/pdf/folder --list
```

### 3. Use with Claude Code (MCP)

Generate the config:

```bash
pdf-index /path/to/pdf/folder --generate-mcp-config
```

Save the output to `.mcp.json` in your project directory. Claude Code will then have access to these tools:

| Tool | Description |
|---|---|
| `search_tables` | Find tables matching keywords, data types, or numeric ranges |
| `get_table` | Get full table data by ID |
| `get_document_summary` | Overview of a PDF (pages, tables, metadata) |
| `get_page_content` | Text + tables from a specific page |
| `list_documents` | All indexed PDFs with stats |
| `query_data` | Run read-only SQL against extracted data |
| `add_observation` | Record research findings in the active session |
| `get_session_notes` | Retrieve session observations |

## CLI Reference

```
pdf-index <path>                  Index all PDFs in directory
pdf-index <path> --list           List indexed documents
pdf-index <path> --search "term"  Search tables by keyword
pdf-index <path> --serve-mcp      Start MCP server
pdf-index <path> --generate-mcp-config  Print .mcp.json snippet
pdf-index <path> --session-start  Start a research session
pdf-index <path> --session-end    End the active session
pdf-index <path> --observe "note" Add an observation
pdf-index <path> --observations   Show session observations
pdf-index <path> -v               Verbose output
```

## How It Works

1. **Scan** — Walks the directory for PDF files
2. **Hash** — SHA-256 hash of each file; skips unchanged documents
3. **Extract** — pdfplumber extracts tables (automatic detection + word-coordinate fallback)
4. **Type** — Cells are classified: currency ($1,234), percent (12.5%), fiscal year (2020-21), number, text
5. **Store** — Everything goes into `.pdfindex/index.db` (SQLite)
6. **Query** — CLI search, MCP tools, or direct SQL

## Windows Notes

Run Python with `-X utf8` to avoid encoding issues:

```bash
python -X utf8 -m pdf_indexer /path/to/pdfs
```

The MCP config generated by `--generate-mcp-config` already includes this flag.
